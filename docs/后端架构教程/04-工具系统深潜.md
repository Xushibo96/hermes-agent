# 第 4 章：工具系统（注册、发现、分发、并发）

## 你将学到什么

- 工具从“定义”到“可被模型调用”的全过程。
- `registry` 与 `model_tools` 的职责分工。
- toolset 作为“能力边界”如何工作。

## 工具生命周期图

```mermaid
flowchart LR
    A[tools/xxx_tool.py\nregistry.register] --> B[tools/registry.py\n保存 ToolEntry]
    B --> C[model_tools._discover_tools()\nimport 触发注册]
    C --> D[get_tool_definitions()\n按 toolset/可用性过滤]
    D --> E[run_agent.py\n传给模型]
    E --> F[模型发起 tool call]
    F --> G[registry.dispatch()\n执行 handler]
    G --> H[tool result 回写 messages]
```

## 核心角色

### `tools/registry.py`

- 维护工具元数据（schema、handler、toolset、check_fn）。
- 提供统一 `dispatch`。
- 统一异常包装，避免工具异常击穿主循环。

### `model_tools.py`

- 负责工具发现（本地/MCP/插件）。
- 负责 toolset 解析与过滤。
- 负责 sync/async 桥接（持久 event loop）。

### `toolsets.py`

- 负责定义“哪些工具一起暴露”。
- 是安全边界与复杂度边界。

## 为什么要用 register + discover

- 新工具可以按模块独立添加。
- 核心编排层不需要硬编码每个工具细节。
- 支持外部扩展（MCP、插件）并复用同一分发链。

## 并发与安全边界

- 部分工具永不并发（人机交互敏感）。
- 只读工具可并发提升吞吐。
- 文件工具要避免路径冲突。

## 新增工具最小步骤

1. 新建 `tools/xxx_tool.py` 并 `registry.register`。
2. 在 `model_tools._discover_tools()` 导入模块。
3. 在 `toolsets.py` 将工具放入正确集合。
4. 测试：可见性、执行返回格式、错误包装。
