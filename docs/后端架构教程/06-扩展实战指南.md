# 第 6 章：扩展实战（新增工具 / 改循环策略）

## 你将学到什么

- 如何以最小风险新增一个工具。
- 如何调整主循环策略而不破坏收敛性。
- 如何做跨入口一致性验证。

## 实战 A：新增工具

### A1. 设计清单

- 工具是否应进入默认 toolset？
- 是否需要环境变量校验？
- 返回结果是否是 JSON 字符串？

### A2. 改动模板

1. `tools/xxx_tool.py`：定义 handler + register。  
2. `model_tools.py`：加入 discover import。  
3. `toolsets.py`：加入目标 toolset。  
4. 测试：
   - `get_tool_definitions` 可见性
   - `dispatch` 执行结果
   - 错误路径格式一致性

### A3. 回归点

- 工具名称冲突。
- schema 与 handler 参数不一致。
- gateway 环境下 check_fn 行为异常。

## 实战 B：调整主循环策略

### B1. 常见改动

- 调整预算规则。
- 调整并发执行上限。
- 调整失败重试策略。

### B2. 改动原则

- 一次只改一个策略面。
- 保持工具调用消息格式不变。
- 确认 budget 耗尽时仍有可解释输出。

### B3. 验证矩阵

| 场景 | 预期 |
|---|---|
| 单工具调用 | 正常收敛 |
| 多工具链路 | 无死循环 |
| 工具异常 | 错误被包装并继续 |
| 预算耗尽 | 返回明确终止语义 |

## 实战 C：配置行为改动

- 同时验证 CLI/Gateway。
- 同时验证默认 profile/自定义 profile。
- 验证 `HERMES_HOME` 路径隔离。
