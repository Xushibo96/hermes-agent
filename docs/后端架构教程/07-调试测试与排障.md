- 常见故障与修复路径。

## 调试总流程图

```mermaid
flowchart TD
    A[症状出现] --> B{属于哪层?}
    B -->|入口层| C[检查 main/cli/gateway 参数与 cwd]
    B -->|编排层| D[检查 run_agent 循环/预算/重试]
    B -->|能力层| E[检查 discover/toolset/dispatch]
    B -->|状态层| F[检查 SessionDB/压缩阈值]
    C --> G[最小复现实验]
    D --> G
    E --> G
    F --> G
    G --> H[修复后回归验证]
```

## 分层排障清单

### 1) 工具没有被模型调用

先查：
- 工具是否注册成功？
- 工具是否进入当前 toolset？
- `check_fn` 是否返回 True？
- schema 是否正确进入 `get_tool_definitions()`？

### 2) 工具调用后循环不收敛

先查：
- 是否持续返回不可解析内容？
- budget 是否被快速耗尽？
- 是否有并发引发状态冲突？

### 3) CLI 与 Gateway 行为不一致

先查：
- 是否工作目录不同？
- 是否配置桥接导致参数差异？
- 是否 profile 环境变量不一致？

## 推荐测试命令

```bash
python -m pytest tests/test_model_tools.py -q
python -m pytest tests/test_cli_init.py -q
python -m pytest tests/tools/ -q
python -m pytest tests/gateway/ -q
```

> 做核心改动前后建议跑全量：`python -m pytest tests/ -q`

## 关键代码摘要

- 工具问题：从 `model_tools.py` 的 discover/definitions 入手。  
- 收敛问题：从 `run_agent.py` 的预算与 tool loop 入手。  
- 长会话问题：从 `context_compressor.py` 阈值与摘要路径入手。  
- 历史检索问题：从 `hermes_state.py` 的 FTS 与写入路径入手。
